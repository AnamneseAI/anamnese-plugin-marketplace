# Goal Planning

Track and support the user's personal goals, life milestones, and aspirations.

## When to Activate

- User mentions a new goal or aspiration
- User discusses progress on an existing goal
- User reflects on life direction or priorities
- User mentions a milestone or achievement

## Goal Setting Best Practices

When helping the user define goals, apply the SMART framework:
- **Specific** -- clarify vague aspirations into concrete outcomes ("get fit" becomes "run a 5K in under 30 minutes")
- **Measurable** -- include a metric or clear completion criteria
- **Achievable** -- validate the goal feels realistic given the user's context
- **Relevant** -- connect the goal to the user's broader life priorities
- **Time-bound** -- encourage setting a target date or review checkpoint

## Breaking Goals into Milestones

Large goals succeed when decomposed:
- Identify 3-5 milestones that mark meaningful progress
- Each milestone should be achievable in 2-4 weeks
- Define the very next action -- the smallest concrete step the user can take today
- Suggest scheduling goal-related work into daily or weekly routines

## Progress and Review

When the user mentions progress on a goal:
- Acknowledge what's been accomplished before discussing what's next
- Help recalibrate timelines if circumstances have changed
- Suggest a regular review cadence (weekly check-in, monthly reflection)
- Note when a goal has been completed or intentionally dropped

## Connecting Goals Across Life Areas

Be ready to capture goals across all life areas and help the user see connections:
- **Health & fitness** -- weight, exercise, nutrition, sleep
- **Learning & growth** -- new skills, courses, books, hobbies
- **Relationships** -- family, friends, community
- **Travel & experiences** -- trips, adventures, bucket list items
- **Financial** -- savings, investments, purchases (defer details to financial-awareness skill)
- **Creative** -- projects, writing, art, music
- **Life milestones** -- moving, career changes, family planning

When goals span multiple areas (e.g., "save money to travel"), help the user see and leverage the connection rather than treating them in isolation.

## Celebrating Progress

- Mark milestones explicitly -- small wins maintain motivation
- Help the user reflect on how far they've come, not just how far is left
- When a goal is reached, prompt reflection: what worked, what would they do differently
